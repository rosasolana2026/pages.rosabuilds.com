API Reference

Deploy static sites from your AI agent with a single HTTP request. Every deploy gets an instant HTTPS URL on a unique subdomain.

Quickstart

Get from zero to a live URL in under 2 minutes.

1. Get your API key

Sign up at pages.rosabuilds.com/dashboard and copy your API key from the Keys section. Keys look like pk_live_xxxxxxxxxxxxxxxx.

2. Bundle your files

Zip or tar your output directory. The archive must contain an index.html at the root.

bash 
# tar.gz (preferred)
tar -czf bundle.tar.gz -C ./dist .

# or zip
zip -r bundle.zip ./dist/.

3. Deploy

$ curl -X POST https://sites.rosabuilds.com/deploy \
  -H "Authorization: Bearer $PAGES_API_KEY" \
  -F "file=@bundle.tar.gz"
import requests, os

resp = requests.post(
    "https://sites.rosabuilds.com/deploy",
    headers={"Authorization": f"Bearer {os.environ['PAGES_API_KEY']}"},
    files={"file": open("bundle.tar.gz", "rb")},
)
print(resp.json()["url"])  # https://k9xm2p.pages.rosabuilds.com
import FormData from 'form-data';
import fs from 'fs';
import fetch from 'node-fetch';

const form = new FormData();
form.append('file', fs.createReadStream('bundle.tar.gz'));

const res = await fetch('https://sites.rosabuilds.com/deploy', {
  method: 'POST',
  headers: { Authorization: `Bearer ${process.env.PAGES_API_KEY}` },
  body: form,
});
const { url } = await res.json();
console.log(url); // https://k9xm2p.pages.rosabuilds.com
f, _ := os.Open("bundle.tar.gz")
body := &bytes.Buffer{}
w := multipart.NewWriter(body)
fw, _ := w.CreateFormFile("file", "bundle.tar.gz")
io.Copy(fw, f)
w.Close()

req, _ := http.NewRequest("POST",
    "https://sites.rosabuilds.com/deploy", body)
req.Header.Set("Authorization", "Bearer "+os.Getenv("PAGES_API_KEY"))
req.Header.Set("Content-Type", w.FormDataContentType())
resp, _ := http.DefaultClient.Do(req)

Authentication

All API requests must include your API key as a Bearer token in the Authorization header.

http 
Authorization: Bearer pk_live_xxxxxxxxxxxxxxxx

Keep your key secret. Never expose it in client-side code or commit it to version control. Use environment variables: export PAGES_API_KEY=pk_live_...

Keys can be revoked and regenerated at any time from the dashboard. You can create multiple keys per account on Pro and Team plans.

Base URL

All API endpoints are served from the AgentHost backend:

BASE https://sites.rosabuilds.com

All responses are JSON. All requests that include a body should use multipart/form-data for file uploads or application/json for metadata-only updates.

POST /deploy

Deploy a bundle of static files. Returns a live HTTPS URL within milliseconds.

POST /deploy

Request parameters

ParameterTypeDescription
file required file Your bundle as .tar.gz or .zip. Must contain index.html at the root. Max size depends on plan (10 MB free, 100 MB pro).
slug string Optional. Custom subdomain slug (e.g. my-report → my-report.pages.rosabuilds.com). Auto-generated if omitted. Must be lowercase alphanumeric + hyphens.
ttl_hours integer How many hours until this site auto-deletes. Default: 24 on free, 0 (permanent) on Pro+. Set to 0 for no expiry.
headers JSON string Optional. JSON object of custom response headers. Example: {"Cache-Control":"max-age=3600","X-Frame-Options":"DENY"}
label string Optional. Human-readable label shown in dashboard and list API. Max 128 chars.

Response fields

url string The live HTTPS URL of the deployed site.
deploy_id string Unique deploy identifier. Use this to fetch, update, or delete the deploy.
slug string The subdomain slug used. Matches the subdomain portion of url.
size_kb integer Total uncompressed size of deployed files in kilobytes.
files integer Number of files in the deploy.
ttl_hours integer Hours until auto-deletion. 0 = permanent.
expires_at string|null ISO 8601 expiry timestamp, or null if permanent.
created_at string ISO 8601 timestamp of when the deploy was created.

GET /sites/:id

Fetch details and analytics for a single deploy.

GET /sites/:deploy_id

DELETE /sites/:id

Immediately delete a deploy. The URL stops serving within seconds. This is permanent — the slug is released.

DELETE /sites/:deploy_id
bash 
curl -X DELETE https://sites.rosabuilds.com/sites/dep_01j8xt9r \
  -H "Authorization: Bearer $PAGES_API_KEY"

PATCH /sites/:id

Update metadata on an existing deploy — change the TTL, label, or custom headers without re-uploading files.

PATCH /sites/:deploy_id
bash 
curl -X PATCH https://sites.rosabuilds.com/sites/dep_01j8xt9r \
  -H "Authorization: Bearer $PAGES_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"ttl_hours": 0, "label": "Q1 report"}'

Bundle formats

We accept two archive formats. Both must contain a valid index.html at the root of the archive.

FormatExtensionContent-TypeNotes
Gzipped tar.tar.gzapplication/gzipPreferred. Smaller files, faster uploads.
ZIP archive.zipapplication/zipWidely supported. No symlinks.

Tip: If your archive contains a single top-level folder (e.g. dist/index.html), we automatically unwrap it. So tar -czf bundle.tar.gz dist/ works fine.

TTL & expiry

The ttl_hours parameter controls auto-deletion. Default behavior by plan:

PlanDefault TTLMin TTLMax TTLPermanent?
Hobby (free)24h1h24hNo
Pro ($9/mo)0 (permanent)1hno limitYes (ttl=0)
Team ($29/mo)0 (permanent)1hno limitYes (ttl=0)

Custom headers

Pass a JSON-encoded object as the headers form field. These headers are sent on every request served from the deploy.

bash 
curl -X POST https://sites.rosabuilds.com/deploy \
  -H "Authorization: Bearer $PAGES_API_KEY" \
  -F "file=@bundle.tar.gz" \
  -F 'headers={"Cache-Control":"max-age=86400","X-Robots-Tag":"noindex"}'

Error codes

Errors return a JSON object with error (machine code) and message (human readable).

json
{
  "error":   "bundle_too_large",
  "message": "Bundle exceeds plan limit of 10 MB. Upgrade to Pro.",
  "docs":   "https://pages.rosabuilds.com/docs#errors"
}

400missing_fileNo file was included in the request.
400invalid_archiveArchive is corrupt or not a supported format.
400no_index_htmlArchive has no index.html at its root.
400slug_takenRequested slug is already in use by another account.
400bundle_too_largeBundle exceeds your plan's size limit.

Agent examples

Claude agent — deploy a report

python 
import anthropic, requests, tarfile, io, os, pathlib

client = anthropic.Anthropic()

def deploy_html(html: str) -> str:
    """Deploy an HTML string and return the live URL."""
    buf = io.BytesIO()
    with tarfile.open(fileobj=buf, mode="w:gz") as tar:
        data = html.encode()
        info = tarfile.TarInfo(name="index.html")
        info.size = len(data)
        tar.addfile(info, io.BytesIO(data))
    buf.seek(0)
    resp = requests.post(
        "https://sites.rosabuilds.com/deploy",
        headers={"Authorization": f"Bearer {os.environ['PAGES_API_KEY']}"},
        files={"file": ("bundle.tar.gz", buf, "application/gzip")},
    )
    return resp.json()["url"]

# Give Claude a deploy tool
tools = [{
    "name": "deploy_page",
    "description": "Deploy an HTML page and return its live URL",
    "input_schema": {
        "type": "object",
        "properties": {"html": {"type": "string"}},
        "required": ["html"]
    }
}]

Shell one-liner

bash 
# Build, bundle, and deploy in one pipeline
npm run build && \
  tar -czf /tmp/deploy.tar.gz -C dist . && \
  curl -s -X POST https://sites.rosabuilds.com/deploy \
    -H "Authorization: Bearer $PAGES_API_KEY" \
    -F "file=@/tmp/deploy.tar.gz" | jq -r .url

Webhooks

Pro and Team accounts can register a webhook URL to receive events on every deploy, deletion, or expiry. Configure from the dashboard.

json — webhook payload
{
  "event":     "deploy.created",
  "deploy_id": "dep_01j8xt9r",
  "url":       "https://k9xm2p.pages.rosabuilds.com",
  "created_at":"2025-03-08T00:14:22Z"
}
